Build a complete, production-ready full-stack web application called InterviewIQ — a real-time AI-powered mock interview platform where an animated AI character conducts a live, adaptive video interview using the candidate's resume, cover letter, and job description. The experience must feel indistinguishable from a real human interview.

Core Experience (Non-Negotiable)
Once the interview starts, the user never clicks anything. The AI character:

Speaks naturally using real-time voice
Listens to the user's spoken responses
Watches the user via webcam (vision analysis)
Adapts every next question based on what was said AND what was seen
Wraps up naturally when done
Auto-delivers feedback without any user action


Tech Stack
Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS, Framer Motion
Backend: Next.js API routes + a standalone WebSocket server (Node.js with ws library)
Avatar: Three.js with a rigged GLTF humanoid model (use ReadyPlayerMe free avatar or a simple stylised Three.js face with lip sync driven by audio amplitude)
TTS (AI voice): ElevenLabs Streaming API — use eleven_turbo_v2 model for lowest latency, stream audio chunks directly to the browser via WebSocket
STT (user voice): Deepgram Nova-2 real-time WebSocket API — open a persistent WebSocket, stream microphone audio, get transcripts back in real time with interim_results: true
Vision: Every 3 seconds, capture a frame from the user's webcam as base64 and send it to Claude's vision API for analysis
AI Brain: claude-opus-4-8 via Anthropic API — orchestrates everything, generates adaptive questions, analyzes vision frames, produces final feedback
State: Zustand for global interview state
DB: Supabase (PostgreSQL) — store sessions, transcripts, feedback
Deploy: Vercel (frontend) + Railway (WebSocket server)

Real-Time Architecture
WebSocket Server (/server/ws.ts)
Single persistent WebSocket connection per interview session. Handles all real-time events:
CLIENT → SERVER events:
- interview:start { sessionId, resumeText, coverLetterText, jobDescription, config }
- audio:chunk { data: ArrayBuffer } — raw microphone PCM audio
- vision:frame { data: base64string } — webcam frame every 3 seconds
- interview:end

SERVER → CLIENT events:
- avatar:speaking:start
- avatar:speaking:end
- audio:tts:chunk { data: ArrayBuffer } — ElevenLabs audio chunk
- transcript:interim { text }
- transcript:final { text }
- vision:analysis { eyeContact, bodyLanguage, confidence, notes }
- ai:thinking — triggers avatar thinking animation
- interview:question { text } — for transcript display
- interview:complete { feedbackJson }
Interview Orchestration Logic (/server/interviewer.ts)
This is the brain. A class InterviewOrchestrator that:

On interview:start:

Builds a rich system prompt from resume + JD + cover letter + config
Sends opening message to ElevenLabs TTS, streams audio back
Opens Deepgram STT WebSocket


On receiving final transcript from Deepgram:

Appends to conversation history
Calls Claude with full conversation history + latest vision analysis
Claude decides: next question OR follow-up probe OR topic shift OR wrap-up
Streams Claude response to ElevenLabs
Streams TTS audio back to client


On vision:frame:

Every 3 seconds sends frame to Claude Vision (separate lightweight call)
Extracts: eye contact (looking at camera?), posture, expression (confident/nervous/confused)
Stores latest vision context, injects into next Claude call as additional context


Adaptive difficulty logic:

Tracks answer quality score per response (Claude rates it internally 1–10)
If 3 consecutive strong answers → escalate difficulty
If answer shows confusion → ask a clarifying follow-up before moving on
If eye contact drops during a specific answer → Claude notes it and may probe that topic



Claude System Prompt Template
You are [Name], a senior [role based on JD] conducting a real job interview for the position of [job title] at [company if mentioned].

Candidate profile:
RESUME: {resumeText}
COVER LETTER: {coverLetterText}
JOB DESCRIPTION: {jobDescription}

Interview config:
- Type: {type}
- Difficulty: {difficulty}
- Personality: {personality}
- Target duration: {duration}

Current vision context: {latestVisionAnalysis}

Rules:
- Speak naturally, like a real human interviewer. No bullet points, no lists — pure conversational speech.
- Ask ONE question or make ONE comment at a time.
- Reference specific things from their resume and cover letter naturally.
- If their last answer was weak, probe it. Don't move on.
- If vision shows confusion, acknowledge it naturally ("Take your time" or rephrase).
- If vision shows low eye contact on a sensitive question, you may naturally revisit it.
- Vary your pacing. Sometimes react before asking the next question. Sometimes be silent for a beat.
- When it is time to end (based on duration and natural flow), close the interview gracefully and naturally.
- After closing say exactly: [INTERVIEW_COMPLETE] — this is a signal token, not spoken.
- You are NOT an AI assistant. You are a human interviewer. Never break character.

Conversation so far:
{conversationHistory}

Latest candidate response: {lastResponse}
ElevenLabs TTS Integration

Use ElevenLabs Streaming API with WebSocket
Voice: choose a professional, calm voice (e.g. Rachel or Adam)
Stream audio chunks in real time back to client
Client plays chunks via Web Audio API in sequence with no gaps
Amplitude of audio stream drives the avatar jaw animation

Deepgram STT Integration

Open Deepgram WebSocket on interview start
Stream raw PCM audio from user's microphone (MediaRecorder or AudioWorklet)
Use params: model: nova-2, interim_results: true, endpointing: 300, utterance_end_ms: 1000
On utterance_end event → treat as final answer → trigger Claude call
Interim results → show in live transcript on client

Vision Analysis (Claude Vision)
Every 3 seconds client sends a base64 webcam frame. Server calls Claude with:
Analyze this video frame of a job interview candidate. In JSON respond with:
{
  "eyeContact": "strong|moderate|weak|none",
  "posture": "confident|neutral|slouched|tense",
  "expression": "calm|nervous|confused|engaged|blank",
  "presentation": "professional|casual|unprofessional",
  "notes": "one brief observation if notable, else null"
}
Respond ONLY with JSON.

Avatar Implementation (Three.js)
Use a ReadyPlayerMe GLB avatar or build a stylised abstract humanoid face:

Load GLB with GLTFLoader
Morph targets for: mouthOpen, eyeBlinkLeft, eyeBlinkRight, jawForward
Idle animation loop: subtle body sway, breathing (chest scale Y), random blinks every 3–6 seconds
Speaking: drive mouthOpen and jawForward morph targets from real-time audio amplitude using Web Audio API AnalyserNode
Thinking: play a slow head tilt tween using Tween.js or GSAP
Listening: subtle forward lean, periodic slow nod (triggered randomly while STT is active)
Three-point lighting: key light, fill light, rim light — clean, professional look
Background: soft gradient or blurred office environment (CSS or Three.js plane with texture)

If ReadyPlayerMe avatar GLB is unavailable, build a stylised abstract face from Three.js primitives (sphere head, cylinder neck, plane eyes) that still does lip sync and blinks. Still looks good, still feels alive.

Design Direction

Theme: Light, editorial, premium. Off-white (#FAFAF8) base. Near-black (#111) typography. Single accent: electric amber (#F59E0B) or cobalt (#2563EB). Pick one and commit.
Fonts: Clash Display or Playfair Display for headings. DM Sans or Instrument Sans for body. Import from Google Fonts or Fontshare.

Critical Behaviour Rules

The avatar must NEVER be silent between turns. If Claude is processing, play a subtle "thinking" audio cue (a soft hum or breath) and trigger the thinking animation.
Audio chunks from ElevenLabs must play gaplessly — buffer and chain them via Web Audio API.
Deepgram interim transcripts show live on screen but never trigger Claude. Only utterance_end or speech_final triggers Claude.
Vision frames are sent every 3 seconds regardless of speaking state. Vision analysis never blocks the conversation flow.
The interview session must be recoverable — save transcript to Supabase every 30 seconds so a page refresh doesn't lose the session.
On mobile: graceful fallback message ("Best experienced on desktop with camera and microphone").
All API keys are server-side only. Never expose to client.
